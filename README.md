# NormDocs — десктопное приложение «Нормативка → отчёт»

Десктопный клиент на Python с графическим интерфейсом: читает **папки** с уже распакованными нормативными документами и вводными данными, собирает из них текст и последовательно вызывает **три потока Langflow** — составление формы отчёта, заполнение по данным и проверка по нормативке.

---

## Технологии

| Область | Стек |
|--------|------|
| Язык | Python 3 |
| GUI | **tkinter** / **ttk** (входит в стандартную поставку Python на Windows) |
| HTTP-клиент | **requests** — Langflow: `run`, создание потоков, `basic_examples`, список потоков для авто-поиска UUID |
| PDF | **PyMuPDF** (`pymupdf`, модуль `fitz`) |
| DOCX | **python-docx** |
| Ввод данных | обход **каталогов** на диске (`pathlib`); архивы в приложении не распаковываются |
| Упаковка в EXE (опционально) | **PyInstaller** — файл `build_normdocs_exe.spec` |

Конфигурация подключения к Langflow может задаваться в интерфейсе или через переменные окружения (см. ниже).

---

## Функционал

1. **Вкладка «Настройки»**  
   Базовый URL Langflow, API-ключ, лимиты объёма текста. UUID потоков вводить не нужно: при запуске отчёта приложение запрашивает `GET /api/v1/flows/` и находит три потока с именами **«NormDocs — 1.…», «NormDocs — 2.…», «NormDocs — 3.…»** (так их создаёт кнопка ниже). Настройки сохраняются в `%USERPROFILE%\.normdocs_langflow_settings.json`.  
   Кнопка **«Создать три потока в Langflow (API)»** вызывает `GET …/basic_examples/`, шаблон (*Basic Prompting*) и трижды `POST …/flows/`. Кнопка **«Проверить потоки на сервере»** проверяет, что набор из трёх потоков виден по API. Опционально: `normdocs_app/assets/normdocs_chat_template.json`, если на сервере нет starter-потоков. Для ручного задания UUID (отладка) можно использовать переменные `LANGFLOW_FLOW_FORM`, `LANGFLOW_FLOW_FILL`, `LANGFLOW_FLOW_VERIFY` в `.env`.

2. **Вкладка «Отчёт»**  
   Две **папки** на диске (уже распакованные): нормативка и вводные. Пути сохраняются в настройках.  
   Запуск: **«1 — Форма» → «2 — Заполнение» → «3 — Проверка»** (по порядку) или сразу **«Выполнить все три»**. Для шагов 2–3 нужны результаты предыдущих шагов в памяти приложения (после шага 1 хранится текст нормативки и форма). В журнале при обходе файлов пишется прогресс (в т.ч. перед долгим чтением PDF/DOCX).

3. **Конвейер обработки**
   - рекурсивный обход файлов в указанных папках и сбор текста в единый корпус с ограничением по длине (с подробным журналом);
   - поддерживаемые расширения для извлечения текста: **.txt**, **.md**, **.csv**, **.json**, **.xml**, **.html**, **.htm**, **.pdf**, **.docx** (старый **.doc** не поддерживается);
   - три последовательных запроса к Langflow; результаты отображаются на вкладках «1. Форма», «2. Заполненный отчёт», «3. Проверка» и в журнале.

4. **Требования к потокам Langflow**  
   В каждом шаблонном потоке — **Chat Input**, модель и **Chat Output**. Имена потоков NormDocs должны совпадать с префиксами из п. 1, чтобы их нашёл API.

---

## Требования

- **Python 3** (рекомендуется отдельное виртуальное окружение только для этого приложения).
- Запущенный **Langflow** (например, локально; порт по умолчанию в настройках — `7860`).
- Архивы (**rar/zip/7z**) распакуйте вручную (например, **7-Zip**) и укажите в приложении **папки** с результатом.
- Для вызовов API в Langflow нужен **API-ключ** (Settings → API Keys).

## Установка зависимостей

Из корня репозитория (после активации виртуального окружения):

```bash
pip install -r requirements-normdocs.txt
```

Файл `requirements-normdocs.txt` включает: `requests`, `pymupdf`, `python-docx`, `rarfile`, `pyinstaller` (последний нужен только для сборки EXE).

### Тесты (pytest)

```bash
pip install -r requirements-dev.txt
pytest
```

По умолчанию выполняются **быстрые** тесты без вызовов Langflow с ключом. Переменные для интеграционных проверок подхватываются из **`.env`** в корне проекта (тот же `LANGFLOW_API_KEY`, что и у приложения). В `tests/conftest.py` файл загружается **при импорте**, чтобы `skipif` на этапе сбора тестов уже видел ключ.

Проверки с живым Langflow (нужен ключ в `.env` и запущенный сервер):

```bash
pytest -m live_langflow --override-ini="addopts="
```

## Запуск приложения

Убедитесь, что текущий каталог — корень проекта **GEO_DOCS** (чтобы Python находил пакет `normdocs_app`).

**Вариант 1 — скрипт-обёртка (удобно для PyInstaller):**

```bash
python run_desktop.py
```

**Вариант 2 — модуль:**

```bash
python -m normdocs_app.main
```

Далее в окне: заполните **Настройки**, сохраните их, на вкладке **Отчёт** укажите **папки** с документами и нажмите **«Выполнить все три потока»**.

---

## Переменные окружения (альтернатива полям в GUI)

| Переменная | Назначение |
|------------|------------|
| `LANGFLOW_BASE_URL` | Базовый URL, например `http://127.0.0.1:7860` |
| `LANGFLOW_API_KEY` | API-ключ |
| `LANGFLOW_FLOW_FORM` | UUID потока «форма отчёта» |
| `LANGFLOW_FLOW_FILL` | UUID потока «заполнение» |
| `LANGFLOW_FLOW_VERIFY` | UUID потока «проверка» |
| `NORMDOCS_MAX_CORPUS_CHARS` | Лимит символов для нормативки (по умолчанию `120000`) |
| `NORMDOCS_MAX_DATA_CHARS` | Лимит символов для вводных (по умолчанию `120000`) |
| `NORMDOCS_REQUEST_TIMEOUT` | Таймаут HTTP-запросов в секундах (по умолчанию `600`) |

Значения из файла настроек и полей окна при работе в GUI имеют приоритет над «зашитыми» умолчаниями в коде; переменные окружения использует класс `AppConfig.from_env()` в `normdocs_app/config.py` — их удобно задавать при запуске из скрипта или CI.

## Сборка исполняемого файла (Windows)

В отдельном лёгком venv с установленными зависимостями из `requirements-normdocs.txt`:

```bash
pyinstaller build_normdocs_exe.spec --noconfirm
```

Удобнее из корня проекта запустить **`build_normdocs.bat`**: он завершает процесс `NormDocsLangflow.exe` (если был запущен из прошлой сборки), ждёт пару секунд и вызывает PyInstaller с `--noconfirm`.

Ожидаемый результат: каталог `dist/NormDocsLangflow/` с **`NormDocsLangflow.exe`**.

Если PyInstaller падает с **`PermissionError` / «Отказано в доступе»** при удалении `dist\NormDocsLangflow`: закройте запущенный **`NormDocsLangflow.exe`**, закройте окно Проводника, открытое на этой папке, и повторите сборку. Реже мешает антивирус — временно исключите каталог проекта или папку `dist` из фоновой проверки.

## Структура пакета

```
normdocs_app/
  main.py              # точка входа приложения
  config.py            # настройки Langflow и лимиты
  workers.py           # фоновый конвейер (папки → текст → три вызова API)
  ui/main_window.py    # окно tkinter
  services/            # текст из файлов, Langflow, payload-ы, создание/поиск потоков; archive_extract — не используется GUI
run_desktop.py         # запуск для разработки и PyInstaller
requirements-normdocs.txt
build_normdocs_exe.spec
```

---

## Примечания

- Таймаут одного запроса к Langflow задаётся лимитом в конфиге (по умолчанию до 10 минут на запрос) — длинные цепочки LLM могут выполняться долго.
- Если текста из папок слишком мало, приложение покажет ошибку (пороги проверяются в `workers.py`).
